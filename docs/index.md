# Home 
![](logo.png){ width="150" align=right}
Welcome to the documentation for the **Berg CTF Platform.**

Berg is a platform for hosting jeopardy-style CTFs.
<br><br>
## Getting started
To try out Berg on your local machine, click [here](getting-started.md)
## Features
<div class="grid cards" markdown>

- :fontawesome-brands-kubernetes: **Built on Kubernetes**<br>Horizontally scalable, able to handle large competitions
- :fontawesome-solid-cube: **Challenge instance support**<br>Run per-player instances with multiple services supported
- :fontawesome-solid-flag: **Dynamic flags**<br>Place per-player flags in challenge instances
- :fontawesome-solid-archive: **Challenge handouts**<br>Ship handouts as static files or OCI images
- :fontawesome-solid-chart-area: **Observability**<br>OTEL metrics to see what's going on
- :fontawesome-solid-code: **Declarative configuration**<br>Everything can be configured in YAML and Helm

</div>

## Authentication
<div class="grid cards" markdown>

- :fontawesome-brands-discord: **Discord**<br>Allow players to authenticate using discord and sync permissions based on roles
- :fontawesome-brands-openid: **Generic OpenID Connect**<br>Integrate with your favorite OIDC provider for full flexibility

</div>
